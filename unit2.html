GIS Architecture
⚙️
Apr'19 Q2a · Nov'19 Q2a · Apr'25 Q2b · Nov'25 Q2e
GIS ke functional components kya hain? Koi bhi 2 explain karo. 🔥 Hot
▼

5 Functional Components:

  • Data Input — GPS, satellite, digitization se data lena
  • Data Storage & Management — Database mein organized store karna
  • Data Manipulation & Analysis — Buffer, overlay, network analysis
  • Data Output — Maps, reports, PDF export karna
  • User Interface — QGIS/ArcGIS jaisa GUI for interaction

Detail — Data Input:

  • Primary: GPS, remote sensing, field survey
  • Secondary: Scanning paper maps, import existing datasets

Detail — Data Storage:

  • DBMS use hota hai (PostGIS, Oracle Spatial)
  • Spatial data (coordinates) + Non-spatial (name, type) dono store
Road ka spatial data = coordinates. Non-spatial = road name "NH48", speed limit 100 kmph.
Diagram mein 5 components ek cycle mein dikhao: Input → Storage → Analysis → Output → UI → back to Input.

DBMS in GIS
🗄️
Apr'19 Q2b · Apr'23 Q2d · Nov'19 Q2e · Apr'25 Q2d · Nov'25 Q2f
GIS mein DBMS use karne ke reasons kya hain? 🔥 Hot
▼
DBMSDatabase Management System — data ko efficiently store, retrieve aur manage karta hai. E.g. PostgreSQL/PostGIS.

5 Main Reasons:

  • Data Sharing — Multiple users ek hi database access kar sakte hain simultaneously
  • Data Integrity — Data accurate aur consistent rahta hai — ek jagah update = everywhere updated
  • Redundancy Control — Same data baar baar store nahi hota — storage waste nahi
  • Security — Access control — sirf authorized users hi data dekh sakein
  • Query Capability — SQL se complex questions: "500m se lambi saari roads kaunsi hain?"
Bina DBMS: 10 departments ke 10 alag Excel files → ek road rename karo to 10 jagah update karo. DBMS se: ek update = sab automatically correct.
🔗
Apr'23 Q2f · Nov'25 Q2b
GIS aur DBMS ko link kaise karte hain?
▼
  • GIS software har spatial feature ko ek unique Feature ID deta hai
  • Same Feature ID DBMS table mein attribute data se linked hoti hai
  • Map pe road click karo → GIS Feature ID se DBMS query → attributes display
Spatial DB: Attribute DB (DBMS): FeatureID | Geometry FeatureID | Name | Length R001 | (coords) R001 | NH48 | 250 km R002 | (coords) R002 | SH4 | 80 km ↑_________↑ (Linked by FeatureID)
Linking = Spatial layer + Attribute table ko ek common key se join karna. Yahi GIS ka core concept hai!

Relational Data Model
🔗
Apr'19 Q2d · Nov'19 Q2d · Apr'23 Q2e · Nov'25 Q2a
Relational data model kya hai? Example ke saath explain karo. 🔥 Hot
▼
Relational ModelData tables (relations) mein store hota hai. Tables rows + columns se bani hain. Tables keys se connected hain.
Table: Districts Table: Roads DistrictID | Name RoadID | Name | DistrictID 1 | Mumbai 101 | WEH | 1 2 | Pune 102 | Expressway | 1 DistrictID = Foreign Key (link between tables) Query: "Mumbai ki saari roads?" → SELECT * FROM Roads WHERE DistrictID = 1;

Key Terms:

  • Primary Key — Har row ka unique ID (DistrictID)
  • Foreign Key — Doosri table ka reference (DistrictID in Roads table)
  • SQL — Data query karne ki language
Spatial querying (Nov'25): "Pune se 10km ke andar kaunse hospitals hain?" = Spatial SQL query!
Spatial Data Infrastructure
🏗️
Apr'19 Q2f · Nov'25 —
Spatial Data Infrastructure (SDI) kya hai?
▼
SDIFramework jo different organizations ke beech geographic data sharing enable karta hai — policies + standards + technology.

5 Components:

  • Data — Road maps, census data, satellite imagery
  • Metadata — Data ke baare mein info (kab banaya, accuracy)
  • Standards — OGC WMS/WFS — sab compatible ho
  • Access Networks — Internet portals (India's Bhuvan)
  • People & Policy — Producers, users, policymakers
ISRO ka Bhuvan portal — satellite images, flood maps, soil data sab free mein ek jagah. Yahi India ka SDI hai!
Spatial Data Capture
📡
Nov'19 Q2b · Apr'25 Q2e · Nov'25 Q2d
Spatial data capture ke techniques kya hain? 📌 Med
▼

Primary (Direct) Capture:

  • GPS Survey — Field mein GPS le ke points mark karna
  • Remote Sensing — Satellite se images (Landsat, IKONOS)
  • LiDAR — Laser se precise 3D elevation data

Secondary (Indirect) Capture:

  • Digitization — Paper map screen pe manually trace karna
  • Vectorization — Scanned map ko automatically vector mein convert karna
  • Existing Databases — Census data, Survey of India maps import

4 Stages of Spatial Data Handling:

  • 1. Capture → 2. Storage → 3. Analysis → 4. Presentation
Swiggy map: Primary = GPS se restaurant locate karo. Secondary = purana paper ka street map scan karke digital banao.

Apr'25 Q2a · Nov'25 Q2c
Software technology hardware se slower kyun develop hoti hai?
▼
  • Hardware doubles in power every ~2 years (Moore's Law)
  • Software development slow hai — testing, debugging, compatibility issues
  • Legacy systems — purane software new hardware pe run nahi karte
  • Complex GIS algorithms ko new hardware ke liye rewrite karna padta hai
  • Training aur adoption time lagta hai
Justify format: Hardware (exponential growth) vs Software (linear growth). GIS ke case mein: GPU powerful hai lekin GIS software abhi bhi CPU-based hai mostly.
