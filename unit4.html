Analytical Functions

▼

4 Classifications:

  • 1. Measurements on Vector Data — Length, Area, Perimeter, Distance between features calculate karna
  • 2. Query Functions — Attribute ya spatial queries: "kaunse areas flood-prone hain?"
  • 3. Overlay Functions — Do ya zyada layers combine karna — Union, Intersect, Clip
  • 4. Neighborhood Functions — Ek feature ke aas-paas ka analysis — Buffer, proximity

Neighborhood Functions
📡
Nov'19 Q4a · Apr'23 Q4a · Nov'25 Q4b
Neighborhood functions kya hain? Koi 4 explain karo. 🔥 Hot
▼
Neighborhood FunctionsEk feature ke surrounding area (neighborhood) ko analyze karne ke functions.
  • Buffer — Feature ke around ek zone banana. E.g. School ke 500m buffer = no commercial zone.
  • Thiessen Polygons (Voronoi) — Har point ke liye nearest area define karna. E.g. Nearest hospital zone.
  • Interpolation — Neighboring points se unknown values estimate. E.g. Rainfall map banana.
  • Focal Statistics — Moving window se each cell ki surrounding cells ka avg/max/min. E.g. Elevation smoothing.
  • Slope/Aspect — Terrain ke neighbors se slope aur direction calculate.
Jio tower 3km signal range → Buffer = 3km circle. Us zone mein kaun aata hai = neighborhood analysis!
Data Classification
📊
Nov'19 Q4b · Apr'23 Q4c · Apr'25 Q4d · Nov'25 Q4e
Automatic classification ke 2 main techniques kya hain? 🔥 Hot
▼
ClassificationData ko meaningful groups/classes mein divide karna — map pe different colors se dikhane ke liye.

2 Main Types:

  • User Controlled — User manually class boundaries set karta hai. E.g. "0-25 = Low, 25-50 = Medium, 50+ = High rainfall"
  • Automatic Classification — Software khud decide karta hai:

Automatic Classification ke 2 Techniques:

  • Equal Interval — Data range ko equal parts mein divide karo.
    Data: 0–100, 4 classes → 0-25, 25-50, 50-75, 75-100
  • Natural Breaks (Jenks) — Data mein natural clusters dhundho aur wahan break karo.
    Data: 1,2,3,20,21,22,80,81 → Breaks at 3|20 and 22|80
Rainfall map: Equal interval = mechanical division. Natural Breaks = actually where rainfall "jumps" — zyada meaningful!
Other methods: Quantile (equal number of features per class), Standard Deviation (statistical approach).
Overlay Operations
🔀
Apr'19 Q4c · Nov'19 Q4c · Apr'23 Q4b · Apr'25 Q4b · Nov'25 Q4f
Vector overlay operations kya hain? Diagram ke saath explain karo. 🔥 Hot
▼
Vector OverlayDo ya zyada vector layers combine karna — new layer banana jisme dono layers ki information ho.
Layer A (Flood Zone): Layer B (Agricultural Land): ┌──────┐ ┌──────┐ │ │ │ │ │ A │ OVERLAY → │ A∩B │ └──────┘ └──────┘ (Flood prone farmland)

Vector Overlay Operations:

  • Union (A ∪ B) — Dono layers ka combined result (all features from both)
  • Intersect (A ∩ B) — Sirf overlap wala area (common features)
  • Clip — Layer A ko Layer B se cut karo (cookie cutter)
  • Erase/Difference — Layer B wala area Layer A se remove karo
  • Identity — Layer A features + jo B se overlap kare unke attributes add
Flood zone layer + Agricultural land layer → Intersect → "Kaunsi farmland flood zone mein hai?" — disaster management ke liye zaroori!

Raster Overlay operations kaise karte hain? Decision table example. 🔥 Hot
▼
Raster OverlayDo raster grids ke corresponding cells pe math operations karna — add, subtract, AND, OR, CON.

Apr'23 Q4d — Groundwater Level 2025:

R1 (Current Groundwater Level): 3 6 8 2 7 5 4 9 1 Formula: R2 = CON(R1 > 5, 0) Meaning: Jahan R1 > 5, wahan R2 = 0, else unchanged R2 Result: 3 0 0 2 0 0 4 0 1

Apr'25 Q4c — CON with AND:

A (Land Use): B (Rainfall mm): F F A 5 8 3 A F F 7 6 9 Formula: C = CON((A:'F') AND (B:7), 9, 0) Meaning: Jahan A=Forest AND B=7, wahan C=9, else C=0 C Result: 0 0 0 9 0 0

Nov'19 Q4d — Boolean Operations:

R4 = R1 AND (R2 OR R3) → Pehle R2 OR R3 calculate karo → Phir result ko R1 se AND karo → Ek cell pe: 1 AND (0 OR 1) = 1 AND 1 = 1
CON = Conditional function. CON(condition, value_if_true, value_if_false). Decision table mein conditions aur results clearly write karo!
Network Analysis
🕸️
Apr'23 Q4e · Apr'25 Q4e · Nov'25 Q4a
Network Analysis kya hai? Iske types explain karo. 📌 Med
▼
Network AnalysisConnected lines (roads, pipes, rivers) pe based analysis. Nodes (intersections) aur Edges (roads) se network banta hai.

Types:

  • Shortest Path — A to B ka sabse chhota route. E.g. Google Maps navigation.
  • Travelling Salesman Problem — Multiple stops ke saath optimal route. E.g. Courier delivery.
  • Service Area — X minutes/km mein kahan pahuncha ja sakta hai. E.g. Ambulance 5-min coverage area.
  • Closest Facility — Nearest hospital, police station dhundhna.
  • Flow Analysis — Pipe/river network mein flow direction aur capacity.
Ola/Uber = Network Analysis! Driver ka shortest path + traffic = real-time network analysis. Zomato delivery optimization bhi same.
Error & Application Models
⚠️
Nov'19 Q4f · Apr'23 Q4f · Nov'25 Q4c
GIS mein error kaise propagate hoti hai? 📌 Med
▼
Error PropagationEk step ki error agle steps mein transfer aur accumulate hoti hai — final result mein combined error aati hai.

Error Sources:

  • Input Errors — GPS inaccuracy (±5m), digitization mistakes
  • Processing Errors — Projection conversion, rounding
  • Model Errors — Simplified model real world ko perfectly represent nahi karta
Error Chain Example: GPS Error (±2m) ↓ Digitization Error (±3m) ↓ Overlay Operation ↓ Combined Error = √(2² + 3²) = √13 ≈ 3.6m Final map mein ±3.6m tak error ho sakti hai!
Flood zone map: GPS ±2m + Digitization ±3m + Projection ±1m → Final flood boundary ±4m galat ho sakti hai → wrong evacuation decisions!
Error propagation diagram mein ek chain dikhao: Source → Processing → Analysis → Output. Har step pe error add hoti hai.

GIS Application Models ki 5 characteristics kya hain?
▼

5 Characteristics of GIS-based Application Models:

  • Spatial Orientation — Model location-based data use karta hai
  • Integration — Multiple data sources combine karta hai
  • Dynamic — Time ke saath update hota hai
  • Scale Dependent — Different zoom levels pe different detail
  • Decision Support — Policy aur planning decisions ke liye output deta hai
NDMA (National Disaster Management Authority) ka flood model: Location-based (spatial) + multiple data sources (integration) + real-time updates (dynamic) = GIS application model.
